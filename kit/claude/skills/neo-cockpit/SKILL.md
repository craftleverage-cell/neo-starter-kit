# neo-cockpit — 司令塔ダッシュボード

目的: 「どのプロジェクトが、どこで、なぜ止まっているか」を毎朝5分で把握し、今日の一手を決める。

## 手順

1. スキャン実行（読み取り専用・副作用なし）:
   ```bash
   bash ~/.claude/skills/neo-cockpit/scripts/scan.sh
   ```

2. 「アクティブ判定」: 最終コミットが7日以内 or dirty>0 のプロジェクトについて、
   `~/.claude/projects/<ホームパスから自動生成>/memory/`（ディレクトリ名はホームパスから自動生成。実際の名前は `~/.claude/projects/` 配下を確認）の対応する
   `project_*.md` を読み、「現在地」「次の一手」「ブロッカー」を拾う。
   （MEMORY.md をインデックスとして使う。休眠PJのメモリは読まない＝コンテキスト節約）

3. ダッシュボード出力（この形式厳守・全体で60行以内）:

   | PJ | 状態 | 最終活動 | 次の一手 | ブロッカー/判断待ち |
   |---|---|---|---|---|

   状態の凡例: 🟢アクティブ(7日以内) / 🟡停滞(8-30日) / ⚪️休眠(31日+) / ✅運用のみ

4. **今日の推奨3手**を提示する。結論から。各手に理由を1行添える。
   選定基準: ①売上に直結するか ②締切・打合せが近いか ③10分で終わる詰まり解消か

5. **社長の判断が必要な項目**だけを分離して箇条書きにする
   （例: 休眠PJの凍結/再開、外部送信の承認、課金を伴う作業）。

## ルール

- 完全読み取り専用。このスキル内で git commit / ファイル変更 / 外部送信をしない。
- .env / .env.local / 認証情報ファイルには一切触れない。
- 深掘りが必要になったら Explore サブエージェントに委譲する（自分で全ファイルを読まない）。
- infra health に MISSING / DOWN があれば、ダッシュボード冒頭で⚠️警告する（勝手に修復しない）。
- 出力は結論から。長い説明はしない。
