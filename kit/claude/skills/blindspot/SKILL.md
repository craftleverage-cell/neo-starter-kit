# Blindspot — 盲点チェック

社長（非エンジニアの経営者）が新領域に入る前に、「知らないことすら知らないこと（unknown unknowns）」を洗い出すスキル。実作業を始める前の関所として使う。

## 手順

1. **ヒアリング（1〜2問で簡潔に）**
   対象領域と、社長の現在の知識レベル（完全に素人か・隣接領域の経験があるか）を確認する。質問は1問ずつ、簡潔に。

2. **5カテゴリで盲点を調査・列挙**
   - (a) 規制・法律（許認可・業法・個人情報・広告規制など）
   - (b) 料金体系・コスト構造（初期費用・従量課金・隠れコスト・解約条件）
   - (c) 業界慣習・商習慣（支払いサイト・紹介文化・NGとされる行為）
   - (d) 競合の常識・相場観（価格帯・標準的な提供範囲・差別化の定石）
   - (e) 技術的落とし穴（API制限・スケール限界・ベンダーロックイン）

   必要に応じてWeb検索・コードベース探索をSubAgent（Explore / general-purpose）に委譲する。非エンジニアの盲点は技術よりも (a)(b)(c) に潜みやすいので重点的に。

3. **「知らないと何が起きるか」を添える**
   各盲点に、放置した場合の手戻り・損失の具体像（例:「後からLP全面作り直し」「月額コストが想定の3倍」）を必ず付ける。

4. **重要度順に並べる**
   全体を重要度順に整理し、上位3つは「今すぐ確認すべきこと」として明示する。

5. **指示のコツを提案**
   最後に「この領域でNeoに指示するときのコツ」（何を伝えれば精度が上がるか）を3行で提案する。

6. **出力は平易な日本語**
   非エンジニア向け。専門用語は「つまり○○」で翻訳する。

## 出力フォーマット（目安）

```
## 盲点チェック: [領域名]

### 今すぐ確認すべき Top 3
1. ...（知らないと: ...）
2. ...
3. ...
```
