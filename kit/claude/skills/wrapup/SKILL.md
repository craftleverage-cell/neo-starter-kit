# wrapup — セッション終了の儀式

目的: 次のセッション（または neo-cockpit）が5分で現在地に立てるように、「意味」を記録して終わる。
Obsidianフックは「何をしたか」の生ログを自動記録するが、このスキルは「何を決めて、次に何をするか」を記録する。役割が違う。

## 手順

1. 対象プロジェクトを特定する:
   - このセッションで主に触った `~/ClaudeCode/<name>` のプロジェクト。
   - 複数触った場合はそれぞれについて記録。プロジェクト外の作業なら手順4へ。

2. 事実収集（読み取り専用）:
   ```bash
   git -C <project> status --porcelain | head -20
   git -C <project> log --oneline -5
   ```

3. メモリ更新:
   - `~/.claude/projects/<ホームパスから自動生成>/memory/MEMORY.md`（ディレクトリ名はホームパスの絶対パスから自動生成される。例: `/Users/yourname` → `-Users-yourname`。実際の名前は `~/.claude/projects/` 配下を確認）で
     対象プロジェクトの既存メモリファイルを探す（例: `project_myproject.md`）。
   - **あれば**そのファイルの冒頭に「現在地」セクションを置き（無ければ作り）、下の
     テンプレ（`templates/progress-entry.md`）でセッション記録を追記する。
   - **なければ** `project_<slug>.md` を新規作成し、MEMORY.md に1行インデックスを追加する。
   - ファイルは120行以内を維持: 3セッションより古い記録は「過去ログ要約」1〜3行に圧縮する。

4. プロジェクト外の作業（戦略検討・調査など）だった場合:
   - 恒久的な決定・学びがあれば該当する memory ファイル（feedback_* / reference_*）を更新。
   - 使い捨ての作業なら記録しない（メモリを汚さない）。

5. ユーザーへの報告（結論から・10行以内）:
   - 記録した内容の要約
   - ⚠️ 未コミット変更が残っている場合は明示（**勝手にコミットしない**）
   - 次セッションの推奨初手を1行

## ルール

- git commit / push はしない（社長が明示的に指示した場合のみ、その指示の中で行う）。
- .env / 認証情報には触れない。
- 相対日付（「今日」「来週」）は絶対日付に変換して記録する。
- 記録は事実ベース。テストが通っていないものを「完了」と書かない。
