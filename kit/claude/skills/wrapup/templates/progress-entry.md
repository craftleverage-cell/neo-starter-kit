# progress-entry テンプレート

メモリファイルに追記するセッション記録の標準フォーマット。
「現在地」はファイル冒頭に1つだけ置き、毎回上書きする。セッション記録はその下に新しい順で積む。

```markdown
## 現在地（YYYY-MM-DD時点）
<プロジェクトの状態を1〜2行。次セッションがこれだけ読めば動ける粒度で>

## YYYY-MM-DD セッション記録
- やったこと: <1〜3行・事実のみ>
- 決定事項: <なぜそう決めたかを含めて。なければ「なし」>
- 次の一手: <具体的に。ファイル名・コマンドまで書けるなら書く>
- ブロッカー/判断待ち: <社長判断が必要なものはそう明記。なければ「なし」>
- 未コミット: <あり(N件)/なし>
```

記入ルール:
- 「完了」はテスト・動作確認の証拠があるものだけ。未検証なら「実装済・未検証」と書く。
- 相対日付禁止。すべて絶対日付。
- 3セッションより古い記録は「過去ログ要約」に圧縮し、ファイルを120行以内に保つ。
