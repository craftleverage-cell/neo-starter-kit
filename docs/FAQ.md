# よくある質問（FAQ）

## 無料で使えますか？

いいえ。Claude Codeの利用にはAnthropicの有料プラン（Pro/Max等）、または従量課金のAPIキーが必要です。このキット自体（ルール・エージェント・スキル・Vaultテンプレート）は無料で公開していますが、動かすための本体である Claude Code が有料です。

## Windowsで動きますか？

動きません。macOS専用です。bashスクリプトの前提・Obsidian連携スクリプト・パス構造がすべてmacOSを前提に作られています。Windows対応の予定は今のところありません。

## 既存の `~/.claude` は壊れませんか？

壊しません。インストーラーは上書きする前に必ず `~/.claude/backups/neo-kit-<日時>/` へバックアップを取ります。既存の `settings.json` がある場合は、さらに `~/.claude/settings.json.pre-neo-kit` としても残るので、差分をすぐ確認できます。心配な場合は、実行前に手動で `~/.claude` フォルダごとコピーしておくとより安心です。

## Obsidianは必須ですか？

必須ではありません。Obsidianを入れなくても、Claude Code本体・ルール・エージェント・スキルは問題なく動きます。ただし、Obsidian Vaultへの自動記録（活動ログ・日次サマリー・プロジェクトごとの自動更新）は無効になります。Vaultフォルダ自体はインストール時に作成されるので、後からObsidianをインストールして開くことも可能です。

## jqは必須ですか？

必須ではありませんが、入れることを強く推奨します。安全フック（危険コマンド検知）・監査ログ・Vault自動記録は、いずれもフックへの入力JSONを `jq` でパースする作りです。`jq` が無い場合、これらのフックはエラーで止まるのではなく黙って何もしない（＝安全機能が無効化された状態で全コマンドが素通しされる）ので注意してください。`install.sh` の依存関係チェックで見つからなければインストールを案内します（`brew install jq`）。

## スキルが動きません

同梱の9スキルは、APIキー不要・追加課金なしで動くものだけに絞ってあります（外部依存は実質 git のみ）。動かない場合は、まず [skills-catalog.md](skills-catalog.md) で該当スキルの前提と発動条件を確認してください。`skill-creator-max` で自作したスキルが動かない場合は、そのスキル自身が前提にしている外部ツールやAPIキーが揃っているかを確認します。

## 制作系・調査系のスキルが欲しい（スキルを増やしたい）

このキットには意図的に含めていません。スライド生成・Web調査・SNS分析・デプロイ自動化のようなスキルは、事業や環境ごとに正解が違うためです。同梱の `skill-creator-max` を起動すると、スキルの企画から設計・テスト・配布までを対話形式でガイドしてくれるので、自分の業務に合わせたスキルを自作してください。それがこのキットの設計思想です。

## アンインストールしたい

自動アンインストーラーは同梱していません（設定ファイルが混ざる前提のツールなので、自動削除はかえって事故のもとになります）。README の「⑧ アンインストール」に手動手順を書いてあります。要点は、バックアップから復元するか、該当ファイルを手動で削除するかのどちらかです。

## 自分の会社情報はどこに書けばいいですか？

Obsidian Vaultの `00_CEO_OS/CEO.md` です。会社概要・事業一覧・今期の最優先事項・意思決定の価値観を書いておくと、Neoがそれを踏まえて提案するようになります。あなた自身の役割や得意不得意については、同じフォルダの `MyContext.md` に書きます。

## アップデートしたい

```bash
cd neo-starter-kit
git pull
./install.sh
```

再実行すれば、新しいバージョンのルール・エージェント・フックスクリプトが（バックアップを取った上で）上書きされます。スキルとObsidian Vaultの中身は上書きされません（既存ファイルを残す no-clobber 方式）。
