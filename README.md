# Neo Starter Kit

Claude Code + Obsidian を「AI参謀 Neo」環境として一発でセットアップするスターターキット。

## ① これは何？

Claude Code（AIエージェント型のターミナルツール）と Obsidian（ノートアプリ）を組み合わせて、"AI参謀 Neo" と一緒に会社を運営するための環境を、コマンド1つでセットアップします。
人格定義・安全装置（ガードレール）・エージェント・Obsidian連携・「仕事の回し方」のスキル9本がまとめて入っています。
非エンジニアの経営者が、自分の言葉で話しかけるだけでAIに実務を任せられる状態を目指しています。

> **このキットは、環境と「仕事の回し方」のフレームワークだけを提供します。** スライド生成・Web調査・SNS分析のような制作系・調査系・業務ナレッジ系のスキルは含まれません。それらは事業ごとに正解が違うため、同梱の `skill-creator-max` を使って、各自が自分の業務用スキルを育てていく設計です。

```
┌──────────────────────┐            ┌───────────────────────┐
│   Claude Code (CLI)   │  自動記録   │   Obsidian Vault       │
│   ・Neo(人格)          │ ─────────> │   ・00_CEO_OS           │
│   ・ルール/ガードレール  │            │   ・05_Tickets          │
│   ・エージェント/スキル  │ <───────── │   ・06_Logs             │
└──────────────────────┘   読み込み   └───────────────────────┘
            ▲
            │ ターミナルで話しかける
            │
        あなた（経営者）
```

## ② 必要なもの

| 項目 | 必須度 | 備考 |
|---|---|---|
| macOS 13 (Ventura) 以降 | 必須 | このキットはmacOS専用です |
| **Claude Codeの有料プラン（Pro/Max等）または APIキー** | **必須** | **無料では使えません。** Claude CodeはAnthropicの有料製品です |
| Node.js | 推奨 | Obsidian連携（MCP経由）で使用。無くても主要機能は動きます |
| Obsidian（無料） | 推奨 | ノート保管庫。無くてもClaude Code単体で動きますが、記憶の自動蓄積機能が無効になります |

> 一番見落としやすいポイント: **Claude Codeは無料ツールではありません。** Pro/Maxプランの契約、または従量課金のAPIキーが必要です。契約前提で読み進めてください。

## ③ インストール

### 推奨: git clone

```bash
git clone https://github.com/craftleverage-cell/neo-starter-kit.git
cd neo-starter-kit
./install.sh
```

対話形式でいくつか質問されるので、答えていくだけでセットアップが完了します。すべて既定値で進めたい場合は `./install.sh --yes` を使ってください。

### 別解: curlワンライナー

```bash
curl -fsSL https://raw.githubusercontent.com/craftleverage-cell/neo-starter-kit/main/install.sh | bash
```

> 中身を事前に確認してから実行したい方は、上の `git clone` の方法をおすすめします。シェルスクリプトを `curl | bash` でそのまま実行するのは、ソースを読まずに任意のコードを動かすことと同じだからです。
> なお `install.sh` は自分と同じ場所にある `kit/` フォルダを読みに行く作りなので、リポジトリを手元に置かずにこの一行だけを実行すると「キットのソースディレクトリが見つかりません」というエラーで止まります。その場合は `git clone` の手順に切り替えてください。

### オプション一覧

```
./install.sh --help
```

主なオプション: `--yes`（既定値で進める）/ `--vault <path>`（Vault保存先を指定）/ `--dry-run`（実行予定の操作だけ表示）/ `--skip-deps`（依存関係チェックを省略）

## ④ 何が入るか

| 種類 | 内容 |
|---|---|
| ルール（`~/.claude/rules/`） | 人格定義・開発規約・SubAgent委譲方針・Supabaseセキュリティ規約・「未知発見」運用規約 |
| エージェント（`~/.claude/agents/`） | コードレビュー・ドキュメント作成・セキュリティ監査・サイト設計分析の4種類のサブエージェント |
| スキル（`~/.claude/skills/`） | 「仕事の回し方」の9本（内訳は下表） |
| フックスクリプト（`~/.claude/hooks/`） | 危険操作のブロック・監査ログ・Obsidian自動記録（詳細は下記⑥⑦） |
| Obsidian Vault テンプレート | 経営OS用のフォルダ構成一式（CEO情報・事業・プロジェクト・チケット・ログ） |

同梱スキル9本の内訳（個別の説明は [docs/skills-catalog.md](docs/skills-catalog.md)）:

| 分類 | スキル | 役割 |
|---|---|---|
| 運用リズム系 | `neo-cockpit` / `wrapup` / `standup` | 朝会で全体を見て始め、夜に決定事項を記録して締める毎日のサイクル |
| 計画系 | `devplan` / `grilling` / `blindspot` | 作る前に計画を立て、問い詰めて弱点を炙り、盲点をチェックする |
| 品質系 | `verify` / `security-audit` | 出す前の検証ループと、Claude Code環境自体の安全監査 |
| 拡張 | `skill-creator-max` | 自分の業務用スキルを自作するための対話型ガイド |

9本すべてAPIキー不要・追加課金なしで動きます（外部依存は実質 git のみ）。制作系・調査系のスキルが欲しくなったら、`skill-creator-max` で自作するのがこのキットの流儀です。

## ⑤ 最初の10分

1. `claude` をターミナルで起動し、ブラウザでログインする
2. Obsidianで、先ほど指定したVaultフォルダを開く
3. `00_CEO_OS/CEO.md` に会社概要と今期の最優先事項を書く（5分でいい。空欄のままでも動くが、書くほどNeoの提案が的確になる）
4. `00_CEO_OS/MyContext.md` に自分の役割・得意不得意を書く
5. Claude Codeで `/neo-cockpit` と打ってみる。全体の状況と「今日の推奨」が1画面に出る。「今日何から手をつければいい？」のような雑な質問でも構わない

## ⑥ 安全設計

このキットは「AIに実務を任せる」ことを前提にしているので、安全装置を最初から組み込んでいます。これはこのキットの差別化ポイントです。

- **機密ファイルは読ませない**: `.env` や `secrets/`、SSH鍵などは、Claude Codeの権限設定で最初から読み取り拒否になっています。中身の確認が必要な場面でも、Claudeは「存在するかどうか」までしか調べず、実際の値の確認はあなた自身に委ねる設計です
- **破壊的コマンドは強制ブロック**: `~/.ssh/`・フックスクリプト自体・LaunchAgentsなど、壊れると被害が大きい場所への書き込み・削除・上書きは、通常の許可設定を上書きしてでもハードブロックされます（`pre-bash-guard.sh`）
- **不審なコード実行パターンも検知**: `curl | sh` のような「取得したコードをそのまま実行する」系のコマンドや、Pythonスクリプト内の危険な呼び出し（`subprocess`・`os.system`等）も検知してブロックします
- **全操作を監査ログに記録**: Claude Codeが実行したBash・ファイル編集・読み取りは `~/.claude/audit/` に日付ごとのログとして残ります。機密ファイルに触れた場合はmacOS通知でも知らせます

過信しないでください。送信・公開・支払いのような重要な操作は、最終確認を挟むように運用することを前提にしています。この節の仕組みは、あくまで「うっかり事故」を防ぐための保険です。

## ⑦ Obsidian連携の仕組み

インストール時に有効になる3本のスクリプトが、意識しなくても記憶を蓄積してくれます。

| スクリプト | いつ動くか | 何をするか |
|---|---|---|
| `vault-activity-log.sh` | Claude Codeが何か操作するたび | その日の活動ログ（`06_Logs/Daily_Briefing/YYYY-MM-DD-activity.md`）に1行追記する |
| `vault-daily-summary.sh` | セッション終了時 | 上の活動ログを人間向けに要約し直し、`06_Logs/Daily_Briefing/YYYY-MM-DD.md` を作る |
| `vault-project-sync.sh` | セッション終了時 | `~/ClaudeCode/` 配下の各プロジェクトで何をしたかを、対応する `03_Projects/<プロジェクト名>.md` の自動更新ゾーンに反映する |

いずれも「あらかじめ用意された自動更新ゾーンにしか書き込まない」「存在しないノートを勝手に作らない」という設計なので、あなたが手で書いたメモが上書きされることはありません。Obsidianを起動していなくても動きます（ただのMarkdownファイル操作なので）。

## ⑧ アンインストール

自動アンインストーラーは同梱していません（設定ファイルが混ざる前提のツールなので、自動削除はかえって事故のもとです）。手動で以下を行ってください。

1. `~/.claude/backups/` にあるインストール前のバックアップから、必要なファイルを復元する（複数回インストールした場合は、一番古いタイムスタンプのフォルダが初回分）
2. 不要であれば以下を削除する
   - `~/.claude/CLAUDE.md` / `SOUL-full.md` / `rules/` / `agents/` / `output-styles/neo.md` / `hooks/` / `settings.json` / `neo-kit.conf`
   - Obsidian Vault フォルダ（インストール時に指定した場所）
   - `~/ClaudeCode/`（開発中のプロジェクトが入っている場合は中身を確認してから）
3. Claude Code本体を削除したい場合は、別途 `claude` 公式のアンインストール手順に従ってください（このキットはClaude Code本体のインストーラーではありません）
